## 1. 第一个游戏

**1.1 知识点**

| 知识点 | 说明 |
|--------|------|
| `input(prompt)` | 接收用户键盘输入，**返回值永远是字符串** |

## 3. 转义字符

**3.1 概念**

转义字符以 `\` 开头，用于表示无法直接输入或显示的字符。

**3.2 完整转义字符表**

| 转义序列 | 名称 | 作用 | 示例 |
|----------|------|------|------|
| `\\` | 反斜杠 | 输出一个字面量 `\` | `"C:\\path"` |
| `\'` | 单引号 | 在单引号字符串中嵌入单引号 | `'It\'s'` |
| `\"` | 双引号 | 在双引号字符串中嵌入双引号 | `"He said \"hi\""` |
| `\a` | 响铃 BEL | 触发系统提示音 | `"\a"` |
| `\b` | 退格 BS | 删除前一个字符 | `"abc\bd"` → `abd` |
| `\n` | 换行 LF | 光标移到下一行 | `"a\nb"` |
| `\t` | 水平制表 TAB | 跳至下一个制表位 | `"a\tb"` |
| `\v` | 垂直制表 VT | 垂直方向制表 | 较少使用 |
| `\r` | 回车 CR | 光标回到行首（覆盖） | `"abc\rX"` → `Xbc` |
| `\f` | 换页 FF | 换到新页 | 终端通常无效果 |
| `\ooo` | 八进制转义 | 八进制编码的字符 | `"\141"` → `a` |
| `\xhh` | 十六进制转义 | 十六进制编码的字符 | `"\x61"` → `a` |

**3.3 涉及 Python 特殊写法**
- 八进制 `\ooo` 和十六进制 `\xhh` 转义属于底层编码知识
- `\r` 覆盖行首的特性可以用来做进度条效果

## 4. 原始字符串

**4.1 概念**

在字符串前加 `r` 或 `R`，使字符串内的所有转义字符失效。

**4.2 语法**

```python
r"字符串内容"
r'字符串内容'
```

**4.3 适用场景**

| 场景 | 普通字符串 | 原始字符串 |
|------|-----------|------------|
| Windows 路径 | `"C:\\Users\\name"` | `r"C:\Users\name"` |
| 正则表达式 | `"\\d+\\w+"` | `r"\d+\w+"` |
| LaTeX 公式 | `"\\frac{a}{b}"` | `r"\frac{a}{b}"` |

**4.4 注意事项**
- 原始字符串**末尾不能是奇数个反斜杠**，例如 `r"C:\"` 会报错
- 本质是语法糖，解释器在编译时处理

**4.5 涉及 Python 特殊写法**
- `r""` 前缀是 Python 独有的原始字符串语法
- 并非真的有"原始"数据类型，解释器仍然按正常字符串处理

## 5. 长字符串

**5.1 概念**

用三重引号（三个单引号或三个双引号）包裹的字符串，可以**跨越多行**。

**5.2 语法**

```python
'''这是长字符串'''

"""这也是长字符串"""
```

**5.3 用法**

| 用途 | 示例 |
|------|------|
| 文档字符串（docstring） | 放在模块/类/函数开头 |
| 多行文本块 | 大段文字说明 |
| 多行注释 | 用 `"""` 代替连续多个 `#` |

**5.4 涉及 Python 特殊写法**
- `"""` 既是字符串字面量，又约定俗成用作 docstring
- 如果未赋值给变量，解释器会忽略它（实际生成字符串对象后丢弃）
- 不同于普通注释 `#`，docstring 可以被 `help()` 和 `__doc__` 访问

## 6. 字符串的加法和乘法

**6.1 字符串运算符**

| 运算符 | 操作 | 示例 | 结果 |
|--------|------|------|------|
| `+` | 拼接（连接） | `'520' + '1314'` | `'5201314'` |
| `*` | 重复 | `'Hi' * 3` | `'HiHiHi'` |

**6.2 对比：数字运算**

| 操作 | 类型 | 结果 |
|------|------|------|
| `520 + 1314` | `int + int` | `1834`（数学加法） |
| `'520' + '1314'` | `str + str` | `'5201314'`（拼接） |
| `'str' + 123` | `str + int` | **TypeError**（不允许） |

**6.3 注意事项**
- `+` 两边必须类型一致（都是字符串或都是数字）
- `*` 必须是一边字符串，一边整数

**6.4 涉及 Python 特殊写法**
- 运算符重载：同一运算符在不同类型上有不同行为（多态）
- `"爱你三千遍！\n" * 3000` 可以快速生成大量重复文本

## 7. 比较运算符

**7.1 基础比较运算符（6个）**

| 运算符 | 含义 | 示例 | 结果 |
|--------|------|------|------|
| `<` | 小于 | `3 < 5` | `True` |
| `<=` | 小于等于 | `3 <= 3` | `True` |
| `>` | 大于 | `5 > 3` | `True` |
| `>=` | 大于等于 | `5 >= 5` | `True` |
| `==` | 等于（比**值**） | `[1,2] == [1,2]` | `True` |
| `!=` | 不等于 | `3 != 5` | `True` |

**7.2 身份运算符（2个）⭐ 重点**

| 运算符 | 含义 | 比较对象 |
|--------|------|----------|
| `is` | 同一对象 | 比较内存地址（`id()` 值） |
| `is not` | 不是同一对象 | 比较内存地址（`id()` 值） |

**7.3 `is` vs `==` 对比**

```python
a = [1, 2, 3]
b = [1, 2, 3]

a == b   # True  — 值相同
a is b   # False — 两个独立对象，地址不同
```

| 对比维度 | `==` | `is` |
|----------|------|------|
| 比较什么 | 值（value） | 对象身份（identity） |
| 底层 | 调用 `__eq__` 方法 | 比较 `id()` |
| 典型用途 | 比较内容是否相同 | 判断是否为同一对象、判断 `None` |

**7.4 Python 小整数缓存 🔑**

Python 启动时会预先缓存 **-5 到 256** 之间的整数对象，所有使用这些值的变量都指向同一个对象：

```python
a = 100; b = 100
a is b   # True   ← 小整数缓存

c = 1000; d = 1000
c is d   # False  ← 超出范围，各自创建新对象
c == d   # True   ← 值仍然相等
```

**7.5 最佳实践**
- 判断 `None`：**始终用 `is`**，不用 `==`
- 比较普通值：用 `==`
- 不要依赖小整数缓存，不同 Python 实现可能范围不同

**7.6 涉及 Python 特殊写法**
- `is`/`is not` 是无法被重载的运算符，比较的是 `id()`
- 小整数缓存是 CPython 的实现细节，PyPy/Jython 行为可能不同
